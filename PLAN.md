# Cubic-Bezier Playground

CSS의 `cubic-bezier()` 값을 자유롭게 조절하고 테스트할 수 있는 웹사이트를 만든다.
사용자는 베지어 곡선의 제어점을 직접 드래그하거나 수치를 입력하면서, 해당 easing 값이 실제 애니메이션에 어떤 영향을 주는지 즉시 확인할 수 있다.

## 목표

이 프로젝트의 목표는 CSS 애니메이션을 만들 때 `cubic-bezier(x1, y1, x2, y2)` 값을 더 직관적으로 이해하고 조정할 수 있는 플레이그라운드를 제공하는 것이다.

일반적으로 `ease`, `ease-in`, `ease-out`, `ease-in-out` 같은 기본 easing 값만 사용하는 경우가 많지만, 실제 UI에서는 더 자연스럽거나 브랜드에 맞는 움직임을 위해 커스텀 cubic-bezier 값이 필요하다. 이 웹사이트는 사용자가 곡선을 직접 만지고, 애니메이션 결과를 눈으로 비교하고, 최종 CSS 값을 복사할 수 있도록 돕는다.

## 핵심 사용자 경험

1. 사용자는 화면 중앙의 그래프에서 cubic-bezier 곡선을 확인한다.
2. 두 개의 제어점 P1, P2를 마우스 또는 터치로 드래그해 곡선을 조절한다.
3. 제어점이 움직일 때마다 `cubic-bezier(x1, y1, x2, y2)` 값이 즉시 업데이트된다.
4. 사용자는 x1, y1, x2, y2 값을 직접 입력해서 정밀하게 조정할 수도 있다.
5. 변경된 easing 값으로 샘플 애니메이션이 재생되어 실제 움직임을 바로 확인할 수 있다.
6. 만족스러운 값을 찾으면 CSS 코드 또는 값만 복사할 수 있다.

## 주요 기능

### 1. Cubic-bezier 그래프 에디터

- 0~1 범위의 좌표계를 가진 정사각형 그래프를 제공한다.
- 시작점은 `(0, 0)`, 끝점은 `(1, 1)`로 고정한다.
- 제어점 P1 `(x1, y1)`, P2 `(x2, y2)`를 드래그할 수 있다.
- 제어점과 시작점/끝점을 연결하는 보조선을 표시한다.
- 현재 곡선을 SVG path 또는 Canvas로 시각화한다.
- 드래그 중에도 곡선과 값이 실시간으로 갱신되어야 한다.

### 2. 수치 입력 패널

- `x1`, `y1`, `x2`, `y2` 값을 각각 입력할 수 있는 필드를 제공한다.
- 입력값은 소수점 단위로 조정 가능해야 한다.
- `x1`, `x2`는 CSS cubic-bezier 규칙에 맞게 0~1 범위로 제한한다.
- `y1`, `y2`는 overshoot/bounce 같은 움직임을 표현할 수 있도록 0~1 밖의 값도 허용한다.
- 그래프 조작과 수치 입력은 항상 동기화되어야 한다.

### 3. 애니메이션 프리뷰

- 현재 cubic-bezier 값이 적용된 샘플 애니메이션을 보여준다.
- 기본 프리뷰는 박스가 좌우로 이동하는 형태로 시작한다.
- 재생 버튼을 누르면 애니메이션을 다시 실행한다.
- easing 변경 시 자동으로 프리뷰를 재생하거나, 사용자가 직접 재생할 수 있게 한다.
- 가능하면 선형 움직임과 현재 easing 움직임을 나란히 비교할 수 있도록 한다.

### 4. 프리셋 제공

자주 사용하는 easing 값을 버튼으로 제공한다.

- `linear`: `cubic-bezier(0, 0, 1, 1)`
- `ease`: `cubic-bezier(0.25, 0.1, 0.25, 1)`
- `ease-in`: `cubic-bezier(0.42, 0, 1, 1)`
- `ease-out`: `cubic-bezier(0, 0, 0.58, 1)`
- `ease-in-out`: `cubic-bezier(0.42, 0, 0.58, 1)`

프리셋을 선택하면 그래프, 입력값, 프리뷰가 모두 즉시 업데이트되어야 한다.

### 5. 코드 복사

- 현재 값을 `cubic-bezier(x1, y1, x2, y2)` 형태로 보여준다.
- 다음과 같은 CSS 예시를 함께 제공한다.

```css
transition-timing-function: cubic-bezier(0.25, 0.1, 0.25, 1);
```

- 사용자는 값만 복사하거나 CSS 한 줄 전체를 복사할 수 있다.
- 복사 성공 시 짧은 피드백 메시지를 보여준다.

## 디자인 원칙

- 전체 디자인은 심플하고 미니멀하게 구성한다.
- 그림자는 일절 사용하지 않는다.
- 설명 문구는 가능한 거의 작성하지 않고, 꼭 필요한 라벨과 값 위주로만 보여준다.
- 색상은 회색조 계열만 사용한다.
- 강조가 필요한 요소도 색상이 아니라 굵기, 테두리, 크기, 간격 차이로 구분한다.
- UI 요소는 장식보다 기능을 우선하며, 불필요한 배경, 카드 효과, 그라디언트, 아이콘 사용을 피한다.
- 그래프, 입력, 프리뷰가 명확히 보이도록 충분한 여백과 정돈된 선을 사용한다.

## 화면 구성

### 데스크톱 레이아웃

- 상단: 프로젝트 제목
- 왼쪽 또는 중앙: cubic-bezier 그래프 에디터
- 오른쪽: 수치 입력, 프리셋, 복사 패널
- 하단: 애니메이션 프리뷰 영역

### 모바일 레이아웃

- 상단: 제목과 설명
- 본문: 그래프 에디터
- 아래: 수치 입력, 프리셋, 복사 버튼
- 마지막: 애니메이션 프리뷰

모바일에서는 그래프 조작이 터치로 자연스럽게 동작해야 한다.

## 데이터 모델

앱의 핵심 상태는 다음 네 개의 숫자다.

```ts
type BezierValue = {
  x1: number;
  y1: number;
  x2: number;
  y2: number;
};
```

이 상태를 기준으로 다음 값을 파생한다.

- CSS 문자열: `cubic-bezier(x1, y1, x2, y2)`
- SVG path 데이터
- 제어점의 화면 좌표
- 애니메이션 프리뷰의 timing function

## 구현 계획

### 1단계: 프로젝트 초기 설정

- 프론트엔드 프로젝트를 생성한다.
- 추천 스택은 Vite + React + TypeScript다.
- 스타일링은 CSS Modules, plain CSS, Tailwind CSS 중 하나를 선택한다.
- 초기 화면에 제목, 설명, 빈 그래프 영역, 입력 패널, 프리뷰 영역을 배치한다.

### 2단계: cubic-bezier 상태 관리 구현

- 기본값을 `ease`에 해당하는 `cubic-bezier(0.25, 0.1, 0.25, 1)`로 설정한다.
- `x1`, `y1`, `x2`, `y2` 값을 단일 상태로 관리한다.
- 입력값 변경 시 상태를 업데이트한다.
- 상태 변경 시 CSS 문자열을 자동으로 다시 계산한다.

### 3단계: 그래프 렌더링 구현

- SVG 기반 그래프를 구현한다.
- 좌표 변환 함수를 만든다.
  - 베지어 좌표 `(0~1)`를 SVG 픽셀 좌표로 변환한다.
  - SVG 픽셀 좌표를 베지어 좌표로 다시 변환한다.
- cubic-bezier 곡선을 SVG `path`로 그린다.
- 시작점, 끝점, P1, P2를 원 형태의 핸들로 표시한다.
- 제어선과 격자선을 표시해 사용자가 곡선을 이해하기 쉽게 한다.

### 4단계: 드래그 인터랙션 구현

- P1, P2 핸들에 pointer 이벤트를 연결한다.
- 마우스와 터치를 모두 지원하기 위해 `pointerdown`, `pointermove`, `pointerup`을 사용한다.
- 드래그 중 SVG 좌표를 베지어 좌표로 변환해 상태를 업데이트한다.
- `x1`, `x2` 값은 0~1 사이로 clamp한다.
- `y1`, `y2` 값은 그래프 표현 범위 안에서는 자유롭게 움직일 수 있게 한다.

### 5단계: 입력 패널 구현

- 네 개의 number input을 만든다.
- 입력값은 소수점 2~3자리 정도로 표시한다.
- 그래프에서 드래그하면 입력 필드 값도 업데이트된다.
- 입력 필드에서 값을 바꾸면 그래프 위치도 업데이트된다.
- 유효하지 않은 입력값에 대해서는 최소한의 안내를 제공한다.

### 6단계: 애니메이션 프리뷰 구현

- 현재 cubic-bezier 값을 CSS custom property 또는 inline style로 적용한다.
- 샘플 박스가 좌우로 이동하는 transition 또는 keyframe animation을 구현한다.
- 재생 버튼을 누르면 애니메이션 클래스를 다시 적용해 재생한다.
- easing을 변경했을 때 현재 값이 프리뷰에 즉시 반영되도록 한다.
- 가능하면 `linear` 기준 애니메이션과 현재 easing 애니메이션을 함께 보여준다.

### 7단계: 프리셋과 복사 기능 구현

- 프리셋 버튼을 추가한다.
- 버튼 클릭 시 해당 cubic-bezier 값으로 상태를 변경한다.
- 현재 CSS 값을 복사하는 버튼을 추가한다.
- Clipboard API를 사용해 복사한다.
- 복사 성공/실패 상태를 사용자에게 짧게 표시한다.

### 8단계: 반응형 UI와 접근성 개선

- 모바일 화면에서 그래프와 입력 패널이 세로로 자연스럽게 배치되도록 한다.
- 버튼과 핸들은 터치하기 충분한 크기로 만든다.
- 입력 필드에는 label을 연결한다.
- 키보드만으로도 수치 입력과 프리셋 선택이 가능해야 한다.
- 색상 대비를 충분히 확보한다.

### 9단계: 테스트와 검증

- 대표 프리셋 값들이 정확히 적용되는지 확인한다.
- 그래프 드래그와 입력 필드가 양방향으로 동기화되는지 확인한다.
- `x1`, `x2`가 0~1 범위를 벗어나지 않는지 확인한다.
- `y1`, `y2`가 0~1 밖의 값에서도 정상적으로 표시되는지 확인한다.
- 복사 버튼이 브라우저에서 정상 동작하는지 확인한다.
- 모바일 터치 조작을 확인한다.

- 더 다양한 프리셋 제공

## 기술적 고려사항

- SVG를 사용하면 제어점, 선, 곡선을 DOM 요소로 다루기 쉬워 드래그 인터랙션 구현이 단순하다.
- CSS cubic-bezier에서 x축 제어점은 0~1 범위여야 유효하므로 `x1`, `x2`는 반드시 제한한다.
- y축 제어점은 0~1 밖의 값도 유효하므로 overshoot를 표현할 수 있게 제한을 느슨하게 둔다.
- 그래프의 시각적 y축은 화면 좌표와 반대이므로 좌표 변환에서 주의해야 한다.
- 입력값과 드래그 조작이 같은 상태를 공유해야 UI가 어긋나지 않는다.
- 프리뷰 애니메이션은 사용자가 값을 바꿀 때마다 과도하게 재시작되지 않도록 UX를 조정할 필요가 있다.

## 예상 컴포넌트 구조

```txt
src/
  App.tsx
  components/
    BezierEditor.tsx
    BezierHandle.tsx
    ValueInputs.tsx
    PresetList.tsx
    AnimationPreview.tsx
    CopyPanel.tsx
  utils/
    bezier.ts
    clamp.ts
  styles/
    global.css
```

각 컴포넌트의 역할은 다음과 같다.

- `App`: 전체 상태를 관리하고 주요 영역을 배치한다.
- `BezierEditor`: SVG 그래프와 드래그 인터랙션을 담당한다.
- `BezierHandle`: 제어점 핸들을 렌더링한다.
- `ValueInputs`: 네 개의 수치 입력 필드를 제공한다.
- `PresetList`: 프리셋 버튼 목록을 제공한다.
- `AnimationPreview`: 현재 easing 값을 적용한 애니메이션을 보여준다.
- `CopyPanel`: CSS 문자열 표시와 복사 기능을 담당한다.
- `bezier.ts`: 좌표 변환과 SVG path 생성 로직을 담는다.
- `clamp.ts`: 숫자 범위 제한 유틸리티를 담는다.
